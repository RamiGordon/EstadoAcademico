OSDE Template Archetype - README
 

General

Este arquetipo genera una estructura multi-módulo con los archivos que conforman el template del look & feel para el cliente web.

Los módulos generados por el arquetipo son los siguientes:

  • academia-backend:  Lógica de negocio e implementación de web-services
  • academia-frontend:  Lógica de vista
  • academia-services:  Interfaces de web-services
  • academia-entities:  Beans referenciados por las interfaces de los web-services

La estructura generada por el arquetipo soporta 2 modos de generación de WARs que se debe elegir dependiendo de la estrategia de deploy de la aplicación. La configuración por default es la de deploy distribuído.


 

Deploy centralizado

Para el deploy centralizado se genera un único WAR que contiene las clases y recursos de todos los módulos bajo la carpeta WEB-INF/classes. Este modelo se debe utilizar cuando el módulo frontend y backend son vistos como una única aplicación.

Pasos a seguir:

1- Verificar que la configuración en el archivo pom.xml del módulo academia-frontend no posea comentada la sección que se muestra a continuación:

<!-- Este plugin debe comentarse si se va a realizar un deploy distribuído -->
<plugins>
    <plugin>
        <groupId>org.codehaus.mojo</groupId>
        <artifactId>build-helper-maven-plugin</artifactId>
            <executions>
             ..
            ..
            </executions>
    </plugin>
</plugins>

2- Verificar que la configuración en el archivo pom.xml del módulo academia-frontend posea comentada la sección que se muestra a continuación:

<!-- Las dependencias a los modulos deben comentarse si
se va a deployar toda la aplicación como un único war -->

 <!--
<dependency>
    <groupId>ar.com.osde</groupId>
    <artifactId>academia-entities</artifactId>
    <version>1.0.0-SNAPSHOT</version>
</dependency>

<dependency>
    <groupId>ar.com.osde</groupId>
    <artifactId>academia-services</artifactId>
    <version>1.0.0-SNAPSHOT</version>
</dependency>
-->

3- Ejecutar mvn clean package en la carpeta academia-frontend

 

Deploy distribuído

Para el deploy distribuído se generan dos WARs uno que contiene todo lo referente para el frontend y el otro todo lo referente al backend. Este modelo se debe utilizar cuando los módulos frontend y backend son vistos como aplicaciones separadas que se comunican a través de algún protocolo remoto.

Pasos a seguir:

1- Verificar que la configuración en el archivo pom.xml del módulo academia-frontend posea comentada la sección que se muestra a continuación:

<!-- Este plugin debe comentarse si se va a realizar un deploy distribuído -->
<!--
<plugins>
    <plugin>
        <groupId>org.codehaus.mojo</groupId>
        <artifactId>build-helper-maven-plugin</artifactId>
            <executions>
            ..
            ..
            </executions>
    </plugin>
</plugins>
-->

2- Verificar que la configuración en el archivo pom.xml del módulo academia-frontend no posea comentada la sección que se muestra a continuación:

<!-- Las dependencias a los modulos deben comentarse si
se va a deployar toda la aplicación como un único war -->

 <dependency>
    <groupId>ar.com.osde</groupId>
    <artifactId>academia-entities</artifactId>
    <version>1.0.0-SNAPSHOT</version>
</dependency>

<dependency>
    <groupId>ar.com.osde</groupId>
    <artifactId>academia-services</artifactId>
    <version>1.0.0-SNAPSHOT</version>
</dependency>

3- Ejecutar mvn install en la carpeta academia
